## Linksee 運用（次セッション以降）

**方針（まー合意 2026-06-25）**: 開発 Cursor セッションでは **Linksee を積極利用**し、handoff / backlog と併せて **状況の更新**を残す。未知の将来需要（MEM-8 教訓）のため、**重複は許容・抜けを避ける**。

| タイミング | やること |
|------------|----------|
| **セッション開始** | `recall({ query: "embodied-claude 2026-06 handoff", entity: embodied-claude-Win })` — チャット履歴は読まない |
| **決定・方針が固まった** | `remember` — layer `learning` / `decision`、importance 高め、必要なら `pinned` |
| **実装完了・本番反映** | `remember` — layer `implementation`、`state: done` |
| **ハマり・偽成功** | `remember` — layer `caveat` |
| **次タスク・未着手** | handoff の「未 / 次」を更新 + Linksee `context` で差分を短く残す |
| **セッション終了** | handoff 1 枚を更新（索引）+ Linksee に「LIVE / ブロッカー解消」を 1 行 |

**置き分け**（Linksee entity `memory-layer-split`）

- **こより LTM**（`:18900` / `memory.db`）— まーの生活 fact、会話で使う記憶
- **Linksee** — リポ開発の決定・ caveat・実装履歴・運用状態
- **profile_gists**（`social.db`）— L0、ベクトル bypass（8e）

MCP: Cursor `~/.cursor/mcp.json` の linksee（**npx 不可** — Cursor 同梱 Node 22 + `~/.config/linksee-mcp`）。
